Test Your Server

Now that you have a potentially working ChatApp, you can proceed to test it. Prior to this you have installed "ws" from the Node Package Manager. If you haven't installed this, then please go back to the Getting Started section to install it.


Host on Local Server

We are going to run the server locally on the command line. To access the command line, reference the 'Finding Command Line' section of the Getting to Know the Command Line tutorial Next, change to directory to where the server file is stored. Proceed to launch your server by typing in the following in the command line:

wscat -l 3000

This will allow the terminal to act as the server for a client to connect. Note that 3000 refers to the port number, so if you used a different number, be sure to replace 3000 with that number when typing the command.


Client Connect

Now that the server is working, you can connect through your browser by simply connecting to the port by entering the following in your address bar:

localhost:3000


This allows you to connect locally through your browser based on the specified port. If the port you selected is not 3000, simply change the port number. Now that your Chat App is running in the browser, you can test the functionality by inputting text.
If the Chat App is not functioning properly, here are some troubleshooting tips:

  • browser console: you can use your browser console to check for errors. To access your browser console, check out: Using the browser console
  • console.log: you can leverage console.log as needed within your code to better understand what is happening. If you use console.log in your client code, the results will appear in your browser console. If you use console.log in your server code, the results will appear in your command line program.
